---
title: "GoodBoyDoc quickstart: install and generate docs fast"
description: "Learn how to install GoodBoyDoc and generate formatted Python documentation from docstrings in under a minute, with Google, NumPy, or Sphinx styles."
---

GoodBoyDoc takes Python source code as input and returns formatted documentation extracted from its docstrings. This guide walks you through installation, your first generation run, and how to switch between docstring styles and output formats.

<Steps>
  <Step title="Install the package">
    Install GoodBoyDoc from PyPI using pip:

    ```bash
    pip install goodboydoc
    ```

    GoodBoyDoc requires Python 3.7 or later and has no required third-party dependencies.
  </Step>

  <Step title="Generate documentation from a Python file">
    Import `generate` and `format_output`, then pass your source code as a string:

    ```python quickstart.py
    from goodboydoc import generate, format_output

    source = open("mymodule.py").read()
    docs = generate(source, style="google")
    print(format_output(docs, fmt="markdown"))
    ```

    `generate` scans `mymodule.py` for triple-quoted docstrings and returns them as a single normalized string, with blocks separated by blank lines.
  </Step>

  <Step title="Try different docstring styles">
    Set the `style` parameter to match your project's docstring convention:

    <Tabs>
      <Tab title="Google">
        ```python
        docs = generate(source, style="google")
        ```

        Google-style docstrings use indented sections like `Args:`, `Returns:`, and `Raises:`. This is the default.
      </Tab>
      <Tab title="NumPy">
        ```python
        docs = generate(source, style="numpy")
        ```

        NumPy-style docstrings use section headers underlined with dashes (`---`). GoodBoyDoc strips the underline separators during normalization.
      </Tab>
      <Tab title="Sphinx">
        ```python
        docs = generate(source, style="sphinx")
        ```

        Sphinx-style docstrings use `:param name:`, `:type name:`, `:returns:`, and `:rtype:` directives.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Choose an output format">
    Pass the result of `generate` to `format_output` with your preferred format:

    <Tabs>
      <Tab title="Markdown (default)">
        ```python
        output = format_output(docs, fmt="markdown")
        ```

        Returns the documentation text as-is — ideal for writing to `.md` files or rendering in Markdown-aware tools.
      </Tab>
      <Tab title="HTML">
        ```python
        output = format_output(docs, fmt="html")
        # Returns: <div class='goodboydoc'><p>...</p></div>
        ```

        HTML-escapes the text, splits on blank lines, and wraps each paragraph in `<p>` tags inside a `<div class='goodboydoc'>`.
      </Tab>
      <Tab title="reStructuredText">
        ```python
        output = format_output(docs, fmt="rst")
        ```

        Adds `=` underlines beneath non-indented lines, producing valid reStructuredText section headers.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Parse a single docstring directly">
    Use `parse_docstring` to normalize one docstring at a time without scanning a whole file:

    ```python
    from goodboydoc import parse_docstring

    raw = """
        Compute the sum of two numbers.

        Args:
            a: First number.
            b: Second number.

        Returns:
            The sum of a and b.
    """

    cleaned = parse_docstring(raw, style="google")
    print(cleaned)
    ```
  </Step>
</Steps>
